Data from Exposure to Ideologically Diverse News and Opinion on Facebook

Overview

This archive contains data analysis procedures and aggregate statistics necessary for understanding and replicating the main results of the paper. We also include code to perform the hard-soft news classification procedure, and well as supplementary datasets on ideological homophily and domain-level audience alignment.

Please see the supporting materials for the paper for additional information.

Supplementary data

We include two supplementary datasets which may be used in other research. supplementary_data.R includes sample code for loading this supplementary data into R.

Site alignment

The results in this paper rely on data that characterize URLs based on the the characteristics of individuals who share them. We call this measure "alignment". We compute this measure by first mapping the 500 most common ideological affiliations listed in individuals' profile under the field "political views" to a five-point {-2, -1, 0, 1, 2} numerical representation. Then, for each "hard content" URL (see [1]), we compute the mean affiliation of every U.S. user over 18 who shared that URL who self-reports an affiliation that can be mapped onto the five-point scale. This table summarizes the alignment of URLs for the top 500 most shared domains.

  • domain: domain
  • avg_alignment: Average alignment
  • l2: proportion of shares with very liberal alignment from domain
  • l1: proportion of shares with liberal alignment from domain
  • m: proportion of shares with moderate alignment from domain
  • r1: proportion of shares with conservative alignment from domain
  • r2: proportion of shares with very conservative alignment from domain

The top 500 most shared domains have each been shared at least 44,000 times. This set of domains covers over 80% of the URLs classified as hard content that had been shared by individuals who self-report a political affiliation. All columns are rounded to the first 4 significant digits. Rows are ordered alphabetically.

Note that alignment is not a measure of slant or bias, it only characterizes the audience who shares a URL on Facebook. The measure may capture differences that include topic matter relevance, framing, etc., in addition to any "bias" present. Our intent here is not to characterize media bias, but instead to characterize the kind of content that partisans find important enough to share.

Ideological homophily

homophily_density.csv contains aggregate statistics about the proportion of links to friends of different ideological affiliations, for liberal, moderate, and conservative users. This table represents the (unsmoothed) version of the kernel density plot given in Figure S1.

  • viewer.affiliation: affiliation of an individual in the sampling frame of the study.
  • friend.affiliation: affiliation of friend.
  • frac.friends: bucketed (in increments of 0.01) fraction of friends with the given affiliation.
  • density: proportion of friends within the given bucket.

Classification

We provide code for the training and classification method used to obtain the results. It can be used with any dataset with the data format specified in the scripts. This code requires Python 2.7, as well as pandas, numpy, re, sklearn, and cPickle. Our results use version pandas v0.15.1, numpy v1.8.0, re v2.2.1, sklearn v0.14.1, and cPickle v1.71.

FitSVM.py

FitSVM.py reads in data URL metadata and fits a model of whether article/page content is "hard" news or "soft," based on sparse features from the accompanying text snippet (usually a summary of the article/page content). The model employed is a linear support vector classifier implemented in Scikit-learn.

ApplySVM.py

ApplySVM.py reads in data on url-shares and classifies article/page content as "hard" news or "soft," based on sparse features from the accompanying text snippet. This code requires model output from FitSVM.py to vectorize and classify the new content.

Main results

We include the R analysis script and datafile with aggregate statistics for reproducing the main results of the text. See the supporting materials of the paper for additional details.

main_analysis.R

main_analysis.R is R code that reads in the data files (described below) to produce the main results of the paper. This code requires R, as well as the dplyr, ggplot2, and xtable packages. It has been tested with R version 3.1.0, ddply v0.4.1, ggplot v1.0.0, xtable 1.7.

pimpd.Rda

pimpd.Rda summarizes the number of cases in which an individual could potentially be exposed to URLs shared by friends ('potential impressions'), whether they saw these stories in their News Feeds, and/or clicked on the link.

  • viewer_affiliation: {-2, -1, 1, 2}-valued variable indicating the self-reported ideological affiliation of the viewer, corresponding to users that are very liberal, liberal, moderate, conservative, and very very conservative.
  • url_alignment: {-1, 1}-valued variable indicating the ideological alignment bin for the URL shared by a friend.
  • has_vpv: {0, 1}-valued variable which is 1 when a validated viewport view (VPV) was recorded.
  • has_click: {0, 1}-valued variable which is 1 if the user had clicked on the link.
  • min_pos: Integer-valued variable with the position of the story, if a VPV exists. When the story is not viewed, this value is NA.
  • pimp_bucket: Approximate number of friends who shared hard content (as a logarithmically-spaced bucket).
  • freq: Number of observations with the above covariates.
  • num.potential: Number of potential impressions (2^(pimp_bucket/100.0))
  • weight: The observation weight for the row: freq/num.potential.

Note that these potential impressions only represent a subset of all stories seen by users; in particular, it only pertains to the population described in the supporting materials of [1], and only includes URLs classified as "hard content" shared some minimum number of users.

vpvd.Rda

vpvd.Rda contains aggregate summaries of interactions with content that appeared in viewers' News Feeds.

  • viewer_affiliation: {-2, -1, 1, 2}-valued variable indicating the self-reported ideological affiliation of the viewer, corresponding to users that are very liberal, liberal, moderate, conservative, and very very conservative.
  • url_alignment: {-1, 1}-valued variable indicating the ideological alignment bin for the URL shared by a friend.
  • has_vpv: {0, 1}-valued variable which is 1 when a validated viewport view (VPV) was recorded.
  • has_click: {0, 1}-valued variable which is 1 if the user had clicked on the link.
  • min_pos: Integer-valued variable with the position of the story, if a VPV exists. When the story is not viewed, this value is NA.
  • imp_bucket: Approximate number of friends whose content users saw in their News Feeds (as a logarithmically-spaced bucket).
  • freq: Number of observations with the above covariates.
  • n_i: Number of validated viewport views (2^(imp_bucket/100.0)).
  • weight: The observation weight for the row: freq/n_i.
